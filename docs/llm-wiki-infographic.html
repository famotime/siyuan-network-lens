LLM Wiki：让知识像系统一样持续生长

不是每次重找答案，而是把阅读、提问与整理不断沉淀成可维护的个人知识体系。把零散资料变成持续演化的 Wiki，把偶然得到的洞察变成反复复用的知识资产。

传统问答 / RAG

提问时再检索，答案常常从头拼装。

每次都在重新找片段、重新串上下文，知识不会自然长成结构。

关键变化

重点不只是找到资料，而是把资料编译成长期可用的知识。

LLM Wiki

平时就持续整理，问题与答案还能继续回写。

交叉引用、矛盾提示、主题综述与新结论会一直累积，知识因此越用越完整。

三层架构

原始资料层

文章、论文、播客、图片、会议记录等都在这里。它们只读保存，是知识系统的事实来源。

Wiki 层

由 LLM 维护的主题页、概念页、人物页、比较页与综述页。你阅读它，LLM 持续更新它。

规则层

命名、归档、互链、索引、日志与维护流程的约定。它决定 LLM 是聊天机器人，还是稳定的知识维护者。

知识沉淀飞轮

核心不是“每次重新发现”，而是“让已经形成的理解继续增长、继续修正、继续被使用”。

知识不会
每次从零开始 而是越用越完整
收集来源 把文章、记录、书摘与素材纳入体系
提炼整合 摘要、归档、互链、修订已有页面
提问回写 把有价值的比较、分析和洞察存回 Wiki
Wiki 演化 页面网络越来越丰富，理解不断更新
好的答案不该消失在对话历史里，而应该成为下一轮理解的起点。

三类关键操作

Ingest 纳入

新资料进入后，LLM 写摘要、更新相关页面，并同步补齐索引与日志。一份来源可能联动多页内容。

Query 提问

你不是只在问答案，而是在推动知识系统继续长出比较、分析、图表、结论与新页面。

Lint 巡检

定期寻找矛盾、过时内容、孤立页面、缺失概念和待补证据，让 Wiki 在增长中保持健康。

适用场景

个人成长

把目标、日记、播客笔记和文章长期串联，逐步形成对自己的稳定理解。

专题研究

围绕一个议题持续积累资料，让论点、证据与反例不断互相修正。

读书陪伴

章节、人物、主题与情节线同步沉淀，阅读过程本身会生成一套 companion wiki。

团队知识库

会议纪要、项目文档、客户访谈与讨论线程不再散落在多个地方。

导航与分工

index.md

内容目录。先回答“这里有什么”，帮助 LLM 快速定位相关页面。

log.md

变化时间轴。记录最近纳入、提问与巡检，让系统知道它是怎样演化到现在的。

人机协作边界

人负责
选资料、提问题、判断方向、决定哪些认知值得保留。
LLM 负责
摘要、互链、归档、对照、修订和跨页维护这些繁琐工作。

为什么它有效

维护成本接近零

人们放弃 Wiki 往往不是因为不重要，而是因为维护太累。

交叉引用自动补齐

LLM 擅长一次触碰多页内容，把遗漏的连接补回来。

新旧观点持续修正

新资料到来时，旧结论不是被遗忘，而是被更新、被挑战、被标注。

知识资产随时间复利

每次阅读、提问与整理都会让下一次理解更快、更稳、更深入。

本质上，LLM Wiki 是一套由 LLM 维护的 Markdown 知识系统。 你负责判断意义，LLM 负责把意义组织成可长期使用的结构。
