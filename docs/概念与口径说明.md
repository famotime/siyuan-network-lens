# 概念与口径说明

本文用于统一说明“脉络镜（Network lens）”当前版本中的核心概念、统计口径与边界条件。

结论以当前代码实现为准，不按 PRD 理想态做推断。若后续实现变更，应优先以对应源码和测试为准。

相关实现主要分布在：

- `src/analytics/analysis.ts`
- `src/analytics/theme-documents.ts`
- `src/analytics/read-status.ts`
- `src/analytics/summary-details.ts`
- `docs/统计卡片规则与定义说明.md`

## 1. 总体分析前提

### 1.1 分析粒度

当前插件的分析粒度是“文档”。

- 节点是文档。
- 边是文档到文档的聚合引用关系。
- 图分析阶段的连通性、社区、桥接、路径、传播节点均按无向图处理。

这意味着：

- 一条 `A -> B` 会让 `A` 与 `B` 在图中视为连通。
- 同一篇文档内的自引用不计入文档级连接。
- 多个块从同一文档指向同一目标文档时，很多统计口径会按文档对去重，而不是按块级引用次数累计。

### 1.2 数据来源

当前引用数据来自两部分：

1. `refs` 表中 `type = 'ref_id'` 的正式引用记录。
2. markdown fallback 解析出的内部链接引用。

当前 fallback 支持：

- `siyuan://blocks/<id>`
- `((block-id "title"))`
- `((block-id 'title'))`

### 1.3 时间窗口与样本

所有分析都先经过统一样本筛选，再进入具体统计。

统一筛选条件包括：

- 笔记本
- 标签
- 关键词
- 主题

时间窗口规则：

- `all`：保留命中筛选条件的全部文档。
- `3d / 7d / 30d / 60d / 90d`：
  - 保留当前窗口内存在有效引用连接的文档；
  - 同时保留当前窗口内最近创建或更新、即使还没有连接的文档。

因此，“最近编辑但暂时没连出去的文档”仍可能出现在文档样本、孤立文档等结果中。

## 2. 基础概念

### 2.1 文档样本

文档样本不是全库文档数，而是“本次分析真正参与计算的文档集合”。

它由以下因素共同决定：

- 当前筛选条件
- 当前时间窗口
- 当前窗口内是否存在有效连接
- 当前窗口内是否最近创建或更新

### 2.2 活跃关系

活跃关系表示当前窗口内真正存在的文档级连接规模。

当前口径是：

- 按唯一有向文档边计数。
- `A -> B` 和 `B -> A` 算两条不同关系。
- 同一文档中多个块重复引用同一目标文档，只算一条 `A -> B`。

它不等于原始引用记录条数。

### 2.3 引用证据

引用证据指某篇文档为什么会被判定为“被引用”或进入相关详情的原始引用记录。

当前实现会为目标文档保留当前窗口内的入链证据，并按来源更新时间倒序排列，便于解释“谁在引用它、最近何时引用”。

## 3. 与主题相关的概念

### 3.1 主题文档

主题文档是“配置驱动识别”的概念，不是自动社区分析结果。

一个文档要被识别为主题文档，当前必须同时满足：

- 位于设置中指定的笔记本中；
- 位于设置中指定的文档路径下；
- 文档标题满足可选的前缀规则；
- 文档标题满足可选的后缀规则；
- 去掉前后缀后还能得到非空主题名称。

主题名称的取值规则是：

- 先读取文档标题；
- 再去掉配置中的主题前缀；
- 再去掉配置中的主题后缀；
- 剩余文本经 `trim()` 后作为主题名。

例如：

- 标题：`主题-图论`
- 前缀：`主题-`
- 后缀：空
- 最终主题名：`图论`

### 3.2 主题

当前实现中的“主题”主要是一个筛选与匹配用的名称值，不是独立图对象。

主题筛选的口径不是“文档是否挂到某个主题文档下面”，而是检查文档的以下字段是否包含所选主题名：

- `title`
- `hpath`
- `path`
- `tags`

因此，主题筛选更像“基于主题名的文本命中筛选”，而不是严格的主题归属关系判断。

### 3.3 主题文档匹配

主题文档匹配主要用于孤立文档的主题建议与补链推荐。

当前会用普通文档的以下字段参与匹配：

- 标题
- 层级路径
- 原始路径
- 标签
- 内容

再与主题文档的以下匹配词进行比对：

- `themeName`
- `name`
- `alias`

匹配结果以命中次数 `matchCount` 表示，用于排序更可能相关的主题文档。

### 3.4 主题社区

主题社区是“图结构分析”概念，不等于主题文档。

当前定义为：

- 基于当前窗口内有效引用关系建立无向图；
- 先识别桥接节点；
- 在去掉桥接节点后，对剩余图做连通分量划分；
- 只保留大小大于 1 的连通分量作为社区。

因此，主题社区表示“自然形成的连接簇”，而不是用户预先定义好的主题页集合。

### 3.5 社区 hub 文档

每个社区会选出最多 3 个 hub 文档。

当前规则是：

- 以该文档在当前邻接图中的连接度 `degree` 从高到低排序；
- 取前 3 个文档作为该社区的 hub 文档。

它表示社区内部更居中的文档，但不等于主题文档。

### 3.6 缺主题页社区 `missingTopicPage`

这是社区上的一个启发式标记，不等于“没有配置主题文档”。

当前判断方式是：

- 如果社区中存在任一文档，其标题或标签包含以下关键词之一：
  - `index`
  - `hub`
  - `map`
  - `overview`
  - `guide`
  - `索引`
  - `导航`
  - `地图`
  - `总览`
- 则认为该社区存在主题页候选；
- 否则记为 `missingTopicPage = true`。

因此：

- `主题文档` 是配置识别；
- `missingTopicPage` 是启发式判断；
- 两者不是同一套逻辑。

## 4. 与结构价值相关的概念

### 4.1 核心文档

核心文档的定义是：

- 当前窗口内至少被 1 个其他文档引用的文档。

它表示“被文档网络关注的中心文档”，不是：

- 编辑最多的文档；
- 出链最多的文档；
- 标签最多的文档。

### 4.2 核心文档排序

当前核心文档排行的排序规则是：

1. `inboundReferences` 倒序
2. `distinctSourceDocuments` 倒序
3. `lastActiveAt` 倒序

但需要注意当前实现细节：

- 当前窗口内，同一来源文档指向同一目标文档的重复块级引用会被去重；
- 因此 `inboundReferences` 与 `distinctSourceDocuments` 在当前实现里通常是同一口径；
- 实际上核心文档榜主要按“有多少个不同文档引用它”排序。

### 4.3 桥接节点

桥接节点不是“连接多就算桥接”，而是“断开后会明显削弱连通性”的文档。

当前实现要求同时满足：

- 它是图中的 articulation point；
- 它的连接度至少为 3；
- 将它移除后，图中至少会出现两个大小都不少于 2 的子组件。

因此，桥接节点强调的是结构性桥梁，而不是普通高频节点。

### 4.4 传播节点

传播节点不是严格图论意义上的 betweenness centrality，而是面向产品解释的启发式结果。

当前计算思路是：

1. 先构造焦点集合，包括：
   - 前 12 个核心文档；
   - 全部桥接节点；
   - 各社区 hub 文档。
2. 对这些焦点两两计算最短路径。
3. 统计哪些中间文档高频出现在这些最短路径上。
4. 再结合：
   - 覆盖了多少焦点文档；
   - 横跨了多少社区；
   - 自身连接度；
   - 是否本身也是桥接节点；
   形成最终结果排序。

它表示“经常出现在关键焦点之间的最短路径上的中间文档”。

## 5. 与断裂和沉没相关的概念

### 5.1 孤立文档

孤立文档的定义是：

- 当前窗口内没有任何有效文档级连接的文档。

也就是该文档在当前窗口内的连接度为 0。

需要强调：

- 只看当前窗口；
- 不看历史上是否曾经连通过；
- 只要当前窗口内有任何有效文档级入链或出链，就不应视为孤立文档。

### 5.2 零散证据 `hasSparseEvidence`

当前孤立文档中，还会额外区分“是否存在少量历史连接痕迹”。

当前规则是：

- 历史上存在连接；
- 历史连接文档数大于 0 且小于等于 2；

则认为该孤立文档存在零散证据。

这表示它不是“从未进入过网络”，而是“曾经只有少量连接，现在在当前窗口里已经断开”。

### 5.3 沉没文档

沉没文档是孤立文档的子集，不是另一个平行概念。

当前规则是：

- 该文档当前仍然是孤立文档；
- 取“最近更新时间/创建时间”与“最后一次历史连接时间”中的较近者；
- 从该时间点开始计算不活跃天数；
- 当不活跃天数达到阈值 `dormantDays` 时，记为沉没文档。

因此，沉没文档强调的是：

- 当前没有连接；
- 且这种无连接状态已经持续足够久。

## 6. 与趋势相关的概念

### 6.1 趋势观察

趋势观察反映的是“文档级连接活跃度变化”，不是“文档编辑频率变化”。

当前比较方式是：

- 当前窗口
- 前一个等长窗口

例如：

- 当前时间窗口是近 7 天，就比较“最近 7 天”与“再往前 7 天”。
- 当前时间窗口是 `all`，则趋势比较固定使用 30 天窗口。

### 6.2 上升文档与回落文档

对每篇文档，分别统计：

- 当前窗口内被多少个不同来源文档引用；
- 前一窗口内被多少个不同来源文档引用。

再计算：

- `delta = currentReferences - previousReferences`

规则：

- `delta > 0`：上升文档
- `delta < 0`：回落文档

### 6.3 新增连接与断裂连接

趋势中的连接变化按无向文档对比较。

也就是说，它看的是：

- 某对文档在当前窗口是否新形成连接；
- 某对文档在前一窗口存在、但当前窗口是否已经断开。

### 6.4 上升社区与沉寂社区

社区趋势的基础是“当前样本文档集合上的社区划分”，然后分别比较这些社区在两个窗口里的内部连接活跃度。

当前判断规则：

- `delta > 0`：上升社区
- `currentReferences === 0`：沉寂社区
- 或 `delta < 0` 且 `currentReferences <= 1`：沉寂社区

## 7. 与已读状态相关的概念

### 7.1 已读文档

已读文档是规则命中结果，不是 AI 语义判断。

当前只要命中以下任一条件，就视为已读：

- 已读标签
- 标题前缀
- 标题后缀
- 已读路径

### 7.2 未读文档

未读文档表示：

- 当前进入样本的文档中；
- 没有命中任何已读规则的文档。

因此，未读并不意味着“从未打开过”，而只是“未命中当前配置的已读规则”。

## 8. 容易混淆的概念区别

### 8.1 主题文档 vs 主题社区

- 主题文档：配置识别出的主题页或主题文档。
- 主题社区：引用网络中自然形成的连接簇。

它们可能重合，但不是同一概念。

### 8.2 主题文档 vs 缺主题页社区

- 主题文档：用户通过设置明确圈定。
- 缺主题页社区：社区内缺少“看起来像索引/导航页”的文档。

前者是配置驱动，后者是启发式判断。

### 8.3 核心文档 vs 社区 hub 文档

- 核心文档：看谁被更多文档引用。
- 社区 hub 文档：看谁在社区结构里连接更多邻居。

一个文档可以同时是两者，也可以只命中其中一个。

### 8.4 孤立文档 vs 沉没文档

- 孤立文档：当前窗口内没有有效连接。
- 沉没文档：在孤立基础上，已经持续不活跃达到阈值。

所以沉没文档一定是孤立文档，但孤立文档不一定是沉没文档。

### 8.5 活跃关系 vs 原始引用数

- 活跃关系：去重后的文档级有向边数量。
- 原始引用数：底层采集到的块级引用记录数量。

当前顶部卡片展示的是前者，不是后者。

## 9. 当前实现边界

为避免误解，下面这些边界当前是成立的：

- 不做块级全量图谱分析，主分析单位是文档。
- 不以黑箱模型定义社区、桥接或传播节点。
- 主题筛选不是严格的主题归属关系推断，而是主题名文本命中。
- 缺主题页社区是启发式信号，不等于主题文档配置结果。
- 传播节点是轻量解释型指标，不等于严格 betweenness centrality。

## 10. 建议的阅读顺序

如果是第一次接触本项目，建议按下面顺序理解：

1. 先看“分析粒度、数据来源、时间窗口”。
2. 再看“主题文档、主题、主题社区”的区别。
3. 再看“核心文档、桥接节点、传播节点”的结构含义。
4. 最后看“孤立文档、沉没文档、趋势观察、已读状态”的业务口径。

这样更不容易把“配置识别”“图结构分析”“统计展示”三层概念混在一起。
