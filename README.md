# 脉络镜 Context lens

脉络镜是一个思源笔记插件，用来帮你看清“文档和文档之间是怎么连起来的”。

它不强调炫目的图谱渲染，而是把引用网络转成更容易行动的结果：哪些文档最核心，哪些主题已经自然形成，哪些内容正在沉没，哪些孤立文档值得补链，哪些文档承担了关键传播和桥接作用。

## 适合什么场景

如果你在思源里已经积累了不少文档，但开始出现下面这些感觉，这个插件就会有用：

- 文档越来越多，但不知道哪些内容是真正的核心入口
- 已经写了很多笔记，却不清楚它们自然形成了哪些主题簇
- 总觉得有些文档写完就沉下去了，之后再也没有进入你的知识网络
- 想整理结构，但不知道应该先补哪些链接，或者该从哪几个主题页入手
- 想从“最近一周”“最近一个月”这样的时间窗口看网络活跃度变化

## 你能看到什么

插件会把当前筛选范围内的文档级引用网络整理成一组可点击的统计卡片和详情列表，帮助你快速定位重点。

当前主要能力包括：

- 核心文档：找出当前窗口内被引用最多、最值得优先维护的文档
- 主题社区：发现自然聚成一组的主题簇，看到它们的规模、标签和 hub 文档
- 孤立文档：找出当前窗口内没有有效文档级连接的内容
- 沉没文档：找出长期没有重新进入网络的内容
- 桥接节点：找出一旦断开，就会削弱多个社区连接的关键文档
- 传播节点：找出频繁出现在关键路径上的中间文档
- 趋势观察：比较当前窗口与前一窗口的活跃变化
- 原始引用证据：查看一篇文档为什么会出现在排行或详情里
- 关系传播路径：查看两个文档之间可能通过哪些中间文档连接

## 它怎么帮你整理笔记

脉络镜不只是“告诉你现状”，还会尽量给出可以直接操作的整理入口。

你现在可以：

- 点击顶部统计卡片，直接查看对应文档详情
- 按时间窗口、笔记本、标签、主题、关键词缩小分析范围
- 拖拽调整顶部统计卡片顺序，让常用指标排在前面
- 在“已读/未读文档”之间切换，快速检查哪些内容还没被标记
- 为孤立文档生成主题链接建议，并一键插入或撤销建议链接

## 主题文档能解决什么问题

如果你的笔记系统里本来就有“主题页 / 索引页 / 专题页”，脉络镜可以把这类文档识别出来，进一步增强整理体验。

配置主题文档后，你会得到：

- 更稳定的“主题”筛选
- 社区分析里更容易识别的主题页
- 针对孤立文档的主题补链建议

这很适合下面这种工作方式：

1. 你先维护一批主题页或索引页
2. 脉络镜帮你找出还没挂回主题网络的孤立文档
3. 你在详情里直接补上建议链接
4. 下次刷新分析时，这些文档就不再是孤立状态

## 已读/未读有什么用

如果你会用标签、标题前缀或标题后缀来标记“已读”，脉络镜可以把这部分规则直接转成统计视图。

你可以用它来：

- 区分哪些文档已经处理过，哪些还没处理
- 从“未读文档”切入，优先做下一轮整理或复习
- 在分析结构的同时，把阅读状态也纳入判断

## 需要知道的识别规则

为了避免误解，下面几条很重要：

- 插件的分析粒度是“文档”，不是普通内容块
- 同一篇文档里的自引用，不算文档级连接
- 孤立文档的判断只看当前时间窗口，不看它历史上是否曾经连通过
- 沉没文档是在“当前不活跃”的前提下，再结合时间阈值得出的结果

当前引用识别来自两部分：

- 思源数据库中的正式引用记录
- markdown 内部链接的 fallback 解析

目前 fallback 已支持：

- `siyuan://blocks/<id>`
- `((block-id "title"))`
- `((block-id 'title'))`

## 如何开始使用

1. 安装插件并打开脉络镜面板
2. 先看顶部统计卡片，找到你现在最关心的入口
3. 用时间窗口、主题、关键词等筛选器缩小范围
4. 点开详情，查看核心文档、孤立文档或社区结构
5. 如果你有主题页，去设置里配置主题文档目录
6. 如果你有阅读流转习惯，顺手配置已读规则

## 安装与构建

如果你是从源码使用这个项目：

```bash
npm install --legacy-peer-deps
npm test
npm run build
```

构建后会更新根目录 `package.zip`，这是正常行为。当前 `plugin.json` 中的最低思源版本为 `3.5.7`。
